## Technical Details

### URL Completion Algorithm

The `_completeUrl()` method uses regex patterns to detect partial URLs:

1. **Already complete** (`/^https?:\/\//i`) - pass through
2. **Port-only** (`/^\d+\//`) - prepend `http://127.0.0.1:`
3. **Colon-port** (`/^:\d+/`) - prepend `http://127.0.0.1`
4. **Hostname/IP** (`/^[\w\.\-]+[\/:/]`) - prepend `http://`
5. **Default** - assume `http://127.0.0.1/`

### Integration Point

The URL completion happens in `_performFetchRequest()` just before calling `fetch()`:

```typescript
// Complete partial URLs before passing to fetch
const url = this._completeUrl(this._uri.toString())

try {
    const response = await fetch(url, fetchOptions)
    // ...
}
```

This ensures all HTTP methods benefit from the feature.

## Usage Examples

### Basic Partial URL
```typescript
import { Http } from 'ejscript'

const http = new Http()
await http.get('4100/index.html')
// Automatically converted to: http://127.0.0.1:4100/index.html
console.log(http.response)
```
